#### Operation symbols, unary relations, binary relations

The unary relation (or "predicate") type is imported from Relation.Unary of the std lib.

```agda
Pred : ‚àÄ {a} ‚Üí Set a ‚Üí (‚Ñì : Level) ‚Üí Set (a ‚äî suc ‚Ñì)
Pred A ‚Ñì = A ‚Üí Set ‚Ñì
```
Sometimes it is useful to obtain the underlying type of a predicate.

\begin{code}

PredType : {A : Type Œ±} ‚Üí Pred A œÅ ‚Üí Type Œ±
PredType {A = A} _ = A

\end{code}

The binary relation types are called `Rel` and `REL` in the standard library, but we
will call them `BinRel` and `BinREL` and reserve the names `Rel` and `REL` for the more
general types of relations we define below and in the Relations.Continuous module.

The heterogeneous binary relation type is imported from the standard library and renamed BinREL.

```agda
BinREL : ‚àÄ {‚Ñì} (A B : Type ‚Ñì) (‚Ñì' : Level) ‚Üí Type (‚Ñì-max ‚Ñì (‚Ñì-suc ‚Ñì'))
BinREL A B ‚Ñì' = A ‚Üí B ‚Üí Type ‚Ñì'
```

The homogeneous binary relation type is imported from the standard
library and renamed BinRel.

```agda
BinRel : ‚àÄ{‚Ñì} ‚Üí Type ‚Ñì ‚Üí (‚Ñì' : Level) ‚Üí Type (‚Ñì ‚äî lsuc ‚Ñì')
BinRel A ‚Ñì' = REL A A ‚Ñì'
```

#### Kernels

The *kernel* of `f : A ‚Üí B` is defined informally by `{(x , y) ‚àà A √ó A : f x = f y}`.
This can be represented in type theory and Agda in a number of ways, each of which
may be useful in a particular context. For example, we could define the kernel
to be an inhabitant of a (binary) relation type, or a (unary) predicate type.

\begin{code}

module _ {A : Type Œ±}{B : Type Œ≤} where

 ker : (A ‚Üí B) ‚Üí BinRel A Œ≤
 ker g x y = g x ‚â° g y

 ker' g I x y = g ‚àò x ‚â° g ‚àò y

 kernel : (A ‚Üí B) ‚Üí Pred (A √ó A) Œ≤

-- The *identity relation* (equivalently, the kernel of a 1-to-1 function)
0[_] : (A : Type Œ±) ‚Üí {œÅ : Level} ‚Üí BinRel A (Œ± ‚äî œÅ)
0[ A ] {œÅ} = Œª x y ‚Üí Lift œÅ (x ‚â° y)

#### Operation type and compatibility

**Notation**. For consistency and readability, we reserve two universe variables for
special purposes.  The first of these is ùìû which shall be reserved for types that
represent *operation symbols* (see [Algebras.Signatures][]). The second is ùì• which we
reserve for types representing *arities* of relations or operations.

In the next subsection, we define types that are useful for asserting and proving
facts about *compatibility* of *operations* with relations, but first we need a
general type with which to represent operations.  Here is the definition, which we
justify below.

The type `Op` encodes the arity of an operation as an arbitrary type `I : Type ùì•`,
which gives us a very general way to represent an operation as a function type with
domain `I ‚Üí A` (the type of "tuples") and codomain `A`. For example, the `I`-*ary
projection operations* on `A` are represented as inhabitants of the type `Op I A` as
follows.

\begin{code}

-- The type of operations on A of arity I
Op : Type Œ± ‚Üí Type ùì• ‚Üí Type (Œ± ‚äî ùì•)
Op A I = (I ‚Üí A) ‚Üí A

-- Example (projections)
œÄ : {I : Type ùì•} {A : Type Œ± } ‚Üí I ‚Üí Op A I
œÄ i x = x i

-- return the arity of a given operation symbol

arity[_] : {I : Type ùì•} {A : Type Œ± } ‚Üí Op A I ‚Üí Type ùì•
arity[_] {I = I} f = I

-- lift a binary relation to the corresponding `I`-ary relation.

eval-rel : {A : Type Œ±}{I : Type ùì•} ‚Üí BinRel A œÅ ‚Üí BinRel (I ‚Üí A) (ùì• ‚äî œÅ)
eval-rel R u v = ‚àÄ i ‚Üí R (u i) (v i)

eval-pred : {A : Type Œ±}{I : Type ùì•} ‚Üí Pred (A √ó A) œÅ ‚Üí BinRel (I ‚Üí A) (ùì• ‚äî œÅ)
eval-pred P u v = ‚àÄ i ‚Üí (u i , v i) ‚àà P


-- If `f : Op I` and `R : Rel A Œ≤`, then we say `f` and `R` are *compatible*
-- just in case `‚àÄ u v : I ‚Üí A`, `Œ† i Íûâ I , R (u i) (v i)  ‚Üí  R (f u) (f v)`.

_preserves_ : {A : Type Œ±}{I : Type ùì•} ‚Üí Op A I ‚Üí BinRel A œÅ ‚Üí Type (Œ± ‚äî ùì• ‚äî œÅ)
f preserves R  = ‚àÄ u v ‚Üí (eval-rel R) u v ‚Üí R (f u) (f v)

--shorthand notation for preserves
_|:_ : {A : Type Œ±}{I : Type ùì•} ‚Üí Op A I ‚Üí BinRel A œÅ ‚Üí Type (Œ± ‚äî ùì• ‚äî œÅ)
f |: R  = (eval-rel R) =[ f ]‚áí R

-- predicate version of the compatibility relation
_preserves-pred_ : {A : Type Œ±}{I : Type ùì•} ‚Üí Op A I ‚Üí Pred ( A √ó A ) œÅ ‚Üí Type (Œ± ‚äî ùì• ‚äî œÅ)
f preserves-pred P  = ‚àÄ u v ‚Üí (eval-pred P) u v ‚Üí (f u , f v) ‚àà P

_|:pred_ : {A : Type Œ±}{I : Type ùì•} ‚Üí Op A I ‚Üí Pred (A √ó A) œÅ ‚Üí Type (Œ± ‚äî ùì• ‚äî œÅ)
f |:pred P  = (eval-pred P) =[ f ]‚áí Œª x y ‚Üí (x , y) ‚àà P


-- The two types just defined are logically equivalent.
module _ {A : Type Œ±}{I : Type ùì•}{f : Op A I}{R : BinRel A œÅ} where

\end{code}
