#### <a id="strongly-well-defined-operations">Strongly well-defined operations</a>

We now describe an extensionality principle that seems weaker than function
extensionality, but still (probably) not provable in [MLTT][]. (We address this
and other questions  below.)  We call this the principle *strong well-definedness
of operations*. We will encounter situations in which this weaker extensionality
principle suffices as a substitute for function extensionality.

Suppose we have a function whose domain is a function type, say, `I ‚Üí A`.  For
example, inhabitants of the type `Op` defined above are such functions.  (Recall,
the domain of inhabitants of type `Op I A := (I ‚Üí A) ‚Üí A` is `I ‚Üí A`.)

Of course, operations of type `Op I A` are well-defined in the sense that equal
inputs result in equal outputs.

#### <a id="questions">Questions</a>

1.  Does the converse `swelldef‚Üífunext` hold or is `swelldef` is strictly weaker
    than `funext`?
2.  If `swelldef` is strictly weaker than `funext`, then can we prove it in MLTT?
3.  If the answer to 2 is no in general, then for what types `I` can we prove
    `swelldef ùì• _ {I}`?

Notice that the implication swelldef ‚Üí funext holds *if* we restrict the universe
level Œ≤ to be `Œπ ‚äî Œ±`. This is because to go from swelldef to funext, we must
apply the swelldef premise to the special case in which `f` is the identify
function on `I ‚Üí A`, which of course has type `(I ‚Üí A) ‚Üí (I ‚Üí A)`.

This is possible if we take `swelldef Œπ Œ± (Œπ ‚äî Œ±)` as the premise (so that we can
assume `B` is `I ‚Üí A`).

It is NOT possible if we merely assume `swelldef Œπ Œ± Œ≤` for *some* Œ≤ (not
necessarily `Œπ ‚äî Œ±`) and for some B (not necessarily `I ‚Üí A`).

In the agda-algebras library, swelldef is used exclusively on operation type, so
that B ‚â° A. I believe there is no way to prove that `swelldef Œπ Œ± Œ±` implies funext Œπ Œ±.


#### <a id="some-new-ideas">Some new ideas</a>

It seems unlikely that we could prove swelldef in MLTT because, on the face of it,
to prove f u ‚â° f v, we need u ‚â° v, but we only have ‚àÄ i ‚Üí u i ‚â° v i.

```agda
swelldef-proof : ‚àÄ {I : Type Œπ}{A : Type Œ±}{B : Type Œ≤}
 ‚Üí                 (f : (I ‚Üí A) ‚Üí B){u v : I ‚Üí A}
 ‚Üí                 (‚àÄ i ‚Üí u i ‚â° v i) ‚Üí f u ‚â° f v
swelldef-proof {I = I}{A}{B} f {u}{v} x = {!!}  --   <== we are stuck
```

However, we *can* prove swelldef in MLTT for certain types at least, using a
zipper argument.

This certainly works in the special case of *finitary* functions, say,
`f : (Fin n ‚Üí A) ‚Üí B` for some `n`.

I expect this proof will generalize to countable arities, but I have yet to
formally prove it.

If `f` is finitary, then we can Curry and work instead with the function

```
(Curry f) : A ‚Üí A ‚Üí A ‚Üí ... ‚Üí A ‚Üí B
```

(for some appropriate number of arrow; i.e., number of arguments).

The idea is to partially apply f, and inductively build up a proof of f u ‚â° f v, like so.

1.     `f (u 0)       ‚â° f (v 0)`                  (by `u 0 ‚â° v 0`),
2.     `f (u 0)(u 1)  ‚â° f (v 0)(v 1)`             (by 1. and u 1 ‚â° v 1),
‚ãÆ
n.     `f (u 0) ‚Ä¶ (u(n-1)) ‚â° f (v 0) ‚Ä¶ (v(n-1))`  (by n-1 and `u(n-1) ‚â° v(n-1)`).
‚ãÆ

Actually, the proof should probably go in the other direction,

‚ãÆ
n.     `f (u 0) ‚Ä¶ (u(n-2))(u(n-1)) ‚â° f (u 0) ‚Ä¶ (u(n-2))(v(n-1))`
n-1.   `f (u 0)   (u(n-2))(u(n-1)) ‚â° f (v 0) ‚Ä¶ (v(n-2))(v(n-1))`
‚ãÆ
2.     `f (u 0)(u 1)  ‚â° f (v 0)(v 1)`
1.     `f (u 0)       ‚â° f (v 0)`


To formalize this, let's begin with the simplest case, that is, when `f : A ‚Üí A
‚Üí B`, so `f` is essentially of type `(Fin 2 ‚Üí A) ‚Üí B`.

However, we still need to establish a one-to-one correspondence between the types
`(Fin 2 ‚Üí A) ‚Üí B` and `A ‚Üí A ‚Üí B`, (and `A √ó A ‚Üí B`), which turns out to be nontrivial.
