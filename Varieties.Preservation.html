### <a id="Equation preservation">Equation preservation</a>

#### <a id="V-is-closed-under-lift">V is closed under lift</a>

As mentioned earlier, a technical hurdle that must be overcome when formalizing proofs in Agda is the proper handling of universe levels. In particular, in the proof of the Birkhoff's theorem, for example, we will need to know that if an algebra ùë® belongs to the variety V ùí¶, then so does the lift of ùë®.  Let us get the tedious proof of this technical lemma out of the way.

Above we proved that `SP(ùí¶) ‚äÜ V(ùí¶)`, and we did so under fairly general assumptions about the universe level parameters.  Unfortunately, this is sometimes not quite general enough, so we now prove the inclusion again for the specific universe parameters that align with subsequent applications of this result.


\begin{code}

#### <a id="S-in-SP">‚®Ö S(ùí¶) ‚àà SP(ùí¶)</a>

Finally, we prove a result that plays an important role, e.g., in the formal proof of Birkhoff's Theorem. As we saw in [Algebras.Products][], the (informal) product `‚®Ö S(ùí¶)` of all subalgebras of algebras in ùí¶ is implemented (formally) in the [UniversalAlgebra][] library as `‚®Ö ùîÑ S(ùí¶)`. Our goal is to prove that this product belongs to `SP(ùí¶)`. We do so by first proving that the product belongs to `PS(ùí¶)` and then applying the `PS‚äÜSP` lemma.

Before doing so, we need to redefine the class product so that each factor comes with a map from the type `X` of variable symbols into that factor.  We will explain the reason for this below.

\begin{code}

module class-products-with-maps {Œ± : Level}
 {X : Type Œ±}
 {feùìïŒ± : funext (ov Œ±) Œ±}
 {fe‚ÇÅ : funext ((ov Œ±) ‚äî (lsuc (ov Œ±))) (lsuc (ov Œ±))}
 {fe‚ÇÇ : funext (ov Œ±) (ov Œ±)}

 ‚Ñë' = Œ£[ ùë® ‚àà (Algebra Œ± ùëÜ) ] ((ùë® ‚àà S{Œ±}{Œ±} ùí¶) √ó (X ‚Üí ‚à£ ùë® ‚à£))

\end{code}
Notice that the second component of this dependent pair type is  `(ùë® ‚àà ùí¶) √ó (X ‚Üí ‚à£ ùë® ‚à£)`. In previous versions of the [UALib][] this second component was simply `ùë® ‚àà ùí¶`, until we realized that adding the type `X ‚Üí ‚à£ ùë® ‚à£` is quite useful. Later we will see exactly why, but for now suffice it to say that a map of type `X ‚Üí ‚à£ ùë® ‚à£` may be viewed abstractly as an *ambient context*, or more concretely, as an assignment of *values* in `‚à£ ùë® ‚à£` to *variable symbols* in `X`.  When computing with or reasoning about products, while we don't want to rigidly impose a context in advance, want do want to lay our hands on whatever context is ultimately assumed.  Including the "context map" inside the index type `‚Ñë` of the product turns out to be a convenient way to achieve this flexibility.


Taking the product over the index type `‚Ñë` requires a function that maps an index `i : ‚Ñë` to the corresponding algebra.  Each `i : ‚Ñë` is a triple, say, `(ùë® , p , h)`, where `ùë® : Algebra Œ± ùëÜ`, `p : ùë® ‚àà ùí¶`, and `h : X ‚Üí ‚à£ ùë® ‚à£`, so the function mapping an index to the corresponding algebra is simply the first projection.

\begin{code}

 ùîÑ' : ‚Ñë' ‚Üí Algebra Œ± ùëÜ
 ùîÑ' = Œª (i : ‚Ñë') ‚Üí ‚à£ i ‚à£

\end{code}

Finally, we define `class-product` which represents the product of all members of ùí¶.

\begin{code}

 class-product' : Algebra (ov Œ±) ùëÜ
 class-product' = ‚®Ö ùîÑ'

\end{code}

If `p : ùë® ‚àà ùí¶` and `h : X ‚Üí ‚à£ ùë® ‚à£`, we view the triple `(ùë® , p , h) ‚àà ‚Ñë` as an index over the class, and so we can think of `ùîÑ (ùë® , p , h)` (which is simply `ùë®`) as the projection of the product `‚®Ö ùîÑ` onto the `(ùë® , p, h)`-th component.

\begin{code}

  where
  lA llA : ‚Ñë' ‚Üí Algebra (ov Œ±) ùëÜ
  lA i =  Lift-Alg (ùîÑ i) (ov Œ±)
  llA i = Lift-Alg (lA i) (ov Œ±)

  slA : ‚àÄ i ‚Üí (lA i) ‚àà S ùí¶
  slA i = siso (fst ‚à• i ‚à•) Lift-‚âÖ

  psllA : ‚àÄ i ‚Üí (llA i) ‚àà P (S ùí¶)
  psllA i = pbase (slA i)

  psPllA : ‚®Ö llA ‚àà P (S ùí¶)
  psPllA = produ psllA

  llA‚âÖA : ‚àÄ i ‚Üí (llA i) ‚âÖ (ùîÑ' i)
  llA‚âÖA i = ‚âÖ-trans (‚âÖ-sym Lift-‚âÖ)(‚âÖ-sym Lift-‚âÖ)

\end{code}
