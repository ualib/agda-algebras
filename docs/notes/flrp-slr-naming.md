# FLRP naming dictionary: the SmallLatticeReps catalog versus this library

This note is the dictionary between two numbering systems that collide in the #483 campaign: the lattice numbering `L1`–`L35` of the DeMeo–Freese–Jipsen manuscript *Representing Finite Lattices as Congruence Lattices of Finite Algebras* (`docs/papers/fin-lat-rep/SmallLatticeReps.tex`, 2016-06-10 draft; upstream `article/SmallLatticeReps.tex` in https://github.com/UniversalAlgebra/fin-lat-rep), and the names this library already uses.  Every certificate module under `src/FLRP/Certificates/SmallLatticeReps/` links here from its header, and every artifact of the #483 campaign states which numbering it uses (issue #485, dictionary task).

## The two warnings

+  **Manuscript `L10` is this library's `L7`** (`Examples.Classical.Lattices.L7`), the distinguished open case of the FLRP program (#484): the seven-element lattice built from the `2 × 3` grid plus one doubly irreducible element.  The unique isomorphism between the two labelings is pinned in § 3 below and enforced by a regression test.
+  **The manuscript's `L7` is a different lattice** — six elements, the chain `0 < 1 < 2 < 4 < 5` with one extra atom `3` under the top — and it *does* have a small representation (`B7`, certified here as `SLR07`).  A bare "`L7`" in FLRP prose is ambiguous; write "library `L7`" or "manuscript `L7`".

More generally, the library's `Examples.Classical.Lattices.L*` names are size-indexed worked examples — library `L2` is the two-element Boolean lattice and library `L3` the three-element Heyting chain — and have nothing to do with the manuscript's catalog numbering (manuscript `L2` is `M3`).  Certificate modules therefore never reuse the bare `L`-names: catalog entry `Ln` becomes module `FLRP.Certificates.SmallLatticeReps.SLRnn` (zero-padded manuscript numbering, e.g. `SLR03` for `L3`/`B3`).

## 1. Sources and conventions

+  The manuscript's § 6 lists the 35 nondistributive, non-ordinal-sum lattices of size ≤ 7, each drawn as a Hasse diagram, most followed by a minimal unary algebra `Bᵢ` with `Con(Bᵢ) = Lᵢ` given by value tables on `{0, …, n−1}`.
+  The element labels used below are the manuscript's own: the tikz node names of each diagram, which run `0` (bottom) to `size − 1` (top) in every entry.  The order is derived mechanically from the manuscript source by `scripts/python/flrp/slr_catalog.py` — nodes carry coordinates, drawn edges are oriented upward by y-coordinate, and the tool rejects any edge that is not a genuine cover of a bounded lattice order — so the cover lists in the table are transcription artifacts, not hand copies.
+  The table below is generated by `python3 scripts/python/flrp/slr_catalog.py --dictionary`; regenerate it there rather than editing it here.
+  Entry status (certified / pending / parked / erratum) is deliberately *not* in this table; that is the census note's job (`docs/notes/flrp-slr-census.md`).

## 3. Manuscript `L10` versus library `L7`, explicitly

Library `L7` uses the numbering of `Examples.Classical.Lattices.L7` (also `docs/notes/flrp-l7-eq6.md` § 1): `0 = ⊥`, `1 = (1,0)`, `2 = (0,1)`, `3 = x`, `4 = (1,1)`, `5 = (0,2)`, `6 = ⊤`.  The unique isomorphism from the manuscript's `L10` labels to the library's `L7` labels is

```text
manuscript  0  1  2  3  4  5  6
library     0  2  1  3  5  4  6
```

that is, the manuscript's atom `1` (the one under two coatoms) is the library's `(0,1)`, its atom `2` is the library's `(1,0)`, and its coatoms `4`, `5` are the library's `(0,2)`, `(1,1)` respectively; `⊥`, `x`, `⊤` coincide.  Uniqueness (library `L7` has no nontrivial automorphisms) and correctness of this table are enforced by `test_slr_catalog.py::test_l10_is_library_l7` against the committed `inputs/l7_lattice.json`.

## 4. A remark on `B2`

The 2016-06-10 draft prints `B2` as the three-element algebra with the single identity operation — a margin note records that R. Freese shrank it from the four-element regular `V4` action so that it really is a smallest representation of `M3` (`Con` of the identity algebra on three points is all of `Eq(3) ≅ M3`).  The older four-element table survives only as a comment in the source, and it is exactly the algebra of the pilot certificate `FLRP.Certificates.Pilot.V4RegularM3` (whose issue-text description "B2 is the regular V4 action" reflects that older reading).  `SLR02` certifies the printed three-element algebra; the pilot stands unchanged as the pipeline's worked example, so `M3` now carries two independent certificates.
