Class-level `S`-closure **fails** (counterexample: `(в„• , +) вҠҶ (в„Ө , +)`, a sub-magma of the magma-reduct of the group `в„Ө` that is not itself a group-reduct), and `H`-closure fails in general too (though it *holds* for the group example, since magma-congruences of a group are group-congruences). ## Conventions to carry

+  **Strict-first, then derive pointwise.**  Whenever `refl` proves a stronger *function-level* `вүЎ` (e.g. `(О» x вҶ’ f вҹЁ$вҹ© x) вүЎ g`), state that first and derive the pointwise corollary (`вҲҖ x вҶ’ f вҹЁ$вҹ© x вүЎ вҖҰ`) by `cong-app`.  Suffix the pointwise one `-ptw`; never use `-вүҲ` for a propositional `вүЎ` result (`-вүҲ` is reserved for genuine setoid-`вүҲ` results).
+  **Equality per level (ADR-006).**  The *signature* category `Sig` uses propositional `_вүЎ_` (laws by `refl`).  The *algebra* category `Alg` uses a **pointwise hom-setoid** `_вүӢ_` (homomorphisms agree up to the codomain's `вүҲ`) вҖ” `_вүЎ_` cannot express this under `--safe` without funext.  The shared `Category` record carries `_вүҲ_` as a field precisely so both instances fit.
+  **Self-contained, no `agda-categories`.**  Keep the minimal `Category` / `Functor` records; extend them (e.g. add `Adjunction` / `Monad` records for M4-5d/e) rather than taking on a dependency.
+  **Placement.**  Signature-level, setoid-free material lives in `Overture`; setoid/algebra-level material in `Setoid/`.  The reduct development is universal algebra and lives in `Setoid/` too: `reduct` is `Setoid.Algebras.Reduct`, `reductF` is `Setoid.Categories.Reduct`, and the variety-level results are `Setoid.Varieties.{Invariance, Reducts}` вҖ” relocated from `Classical/` by ADR-006 (as amended, M4-16 / #403), since they import nothing from `Classical/`.  Only structure-specific category material (`AdjoinUnit`, `Forgetful`) stays in `Classical.Categories`.  (Earlier guidance kept `reduct` in `Classical/` and forbade `Setoid.* вҶ’ Classical.*` imports; that rule was circular and is retired.)

## Inference gotchas (Agda 2.8.0, `--cubical-compatible --exact-split --safe`)

+  **`cong-app` corollaries need their implicits pinned.**  Deriving `pointwise = cong-app strict` often leaves the strict law's implicit signature/setoid unsolved, because the goal is matched through `Carrier (вҹЁ рқ‘Ҷ вҹ© A)` or `o ^ рқ‘Ё` вҖ” and `_^_`, `map`, `вҹҰ_вҹ§` *unfold* through the non-injective `Carrier` / `Interp`, so the unifier cannot invert them.  Two fixes both work: wrap the law and its corollary in a `module _ {рқ‘Ҷ} {A} where вҖҰ` block so the implicits are fixed locally (cleanest вҖ” what `Setoid.Signatures.Functor` does), or pass them explicitly (`cong-app (law {рқ‘Ҷ = рқ‘Ҷ} {A = A}) x`).  The same discipline applies to any lemma whose implicit is a *term* consumed by a recursive function: `вҠ§-reduct`'s equation sides must be pinned at the use site (`{s = вҖҰ} {t = вҖҰ}`), since `s` is not recoverable from `ПҶ вң¶ s` or `вҹҰ s вҹ§` (fourth sighting, M4-5e).
+  **The strict category/functor laws are `refl`, not setoid reasoning.**  The polynomial-functor action is post-composition on the position function, so the coherences reduce by `вҲҳ`-associativity, `id`-cancellation, function-О·, record-О· and ОЈ-О· вҖ” all available under `--cubical-compatible`.  The `Fin n` О·-gap of ADR-002 В§6 does *not* bite here, because these laws compose *abstract* position maps; it bites only when a concrete `Fin`-pattern lambda must be normalized.
+  **Whole-algebra `вүЎ` is not available.**  `reduct id-morphism рқ‘Ё вүЎ рқ‘Ё` cannot be `refl` (the `Interp.cong` proof field would need funext).  State operation- or hom-level equalities instead вҖ” which is exactly why the algebra category's hom-equality is pointwise.
+  **Two-signature functors.**  A functor like `reductF : Functor (Alg рқ‘ҶвӮӮ) (Alg рқ‘ҶвӮҒ)` needs the (`{рқ‘Ҷ}`-parameterized) `Alg` / `hom` / `Algebra` modules at *both* signatures.  Instantiate them by module application inside a `module _ {рқ‘ҶвӮҒ рқ‘ҶвӮӮ} (ПҶ) where`: `private module AвӮҒ = Setoid.Categories.Algebra {рқ‘Ҷ = рқ‘ҶвӮҒ}` and `module AвӮӮ = вҖҰ {рқ‘Ҷ = рқ‘ҶвӮӮ}`.  The `FвӮҒ` action keeps the underlying setoid map and reindexes the source `compatible` through `Оә` (`compatible (projвӮӮ f) {О№ ПҶ o} {a вҲҳ Оә ПҶ o}`, definitionally).
