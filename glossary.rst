    abstract category
      An **abstract category** is one whose objects are not sets or whose morphisms are not functions defined on sets. Our next example is somewhere in between. The objects are sets, but the morphisms are not necessarily *total* functions; that is, they may be defined on only a part of the source object.

    concrete category
      A **concrete category** is one whose objects are sets and whose morphisms are functions defined on these sets (possibly satisfying some other special properties).

    consecutive functions
      If :math:`f : A ‚Üí B` and :math:`g : B ‚Üí C`, then :math:`\mathrm{cod} f = \mathrm{dom} g` and we say that :math:`f` and :math:`g` are **consecutive functions**.

    contravariant powerset functor
      The **contravariant powerset functor** is a functor :math:`P : \mathbf{Set} ‚Üí \mathbf{Set}` such that for each :math:`g : B ‚Üí A` the morphism :math:`g^‚Üê : PA ‚Üí PB` is given by :math:`g^‚Üê (S) = \{b ‚àà B : g(b) ‚àà S\}` for each :math:`S ‚äÜ A`.

    coproduct
      Given two objects :math:`A` and :math:`B` a **coproduct** (or **sum**) of :math:`A` and :math:`B` is denoted by :math:`A+B` and defined to be an object with morphisms :math:`Œπ_1 : A ‚Üí A + B` and :math:`Œπ_2 : B ‚Üí A + B` such that for every object :math:`X` and all morphisms :math:`u : A ‚Üí Y` and :math:`v : B ‚Üí Y` there exists a unique morphism :math:`[u,v] : A+B ‚Üí Y` such that :math:`[u,v] ‚àò Œπ_1 = u` and :math:`[u,v] ‚àò Œπ_2 = v`.

    dependent type
      A **dependent type** is actually a family of types indexed by some parameter. That is, a dependent type provides a *type schema*, which is a collection of types indexed by a set of values. For example, the type ``Fin n`` of finite sets of size ``n`` is a type that *depends* on the value ``n``.  More examples are in :numref:`Section %s <dependent-types>`.  For details, see `the section on Dependent Types <https://leanprover.github.io/theorem_proving_in_lean/dependent_type_theory.html#dependent-types>`_ in the `Theorem Proving in Lean <https://leanprover.github.io/theorem_proving_in_lean/>`_ manual.

    domain
      If :math:`f : A ‚Üí B` is a function or relation from :math:`A` to :math:`B`, then :math:`A` is called the **domain** of :math:`f`, denoted by :math:`\mathrm{dom} f`.

    endofunctor

    endomorphism
      A morphism :math:`f : A ‚Üí A` (i.e., :math:`\mathrm{src} f = \mathrm{tar} f`) is called an **endomorphism**.

    epimorphism
      A morphism :math:`f: A ‚Üí B` is called an **epimorphism** if for every object :math:`Y` and pair :math:`y_1, y_2: B ‚Üí Y` of morphisms, :math:`y_1 ‚àò f = y_2 ‚àò f` implies :math:`y_1 = y_2`. When :math:`f: A ‚Üí B` is an **epimorphism** we often say ":math:`f` is epi" and write :math:`f: ‚Ü† B`.

    equivalence relation
      An **equivalence relation** is a :term:`symmetric` :term:`preorder`. The collection of all equivalence relations on :math:`X` is denoted by :math:`\mathrm{Eq}(X)`.

    equivalent categories
      Two categories :math:`\mathcal C` and :math:`\mathcal D` are called **equivalent categories** if there are functors :math:`F : \mathcal C ‚Üí  \mathcal D` and :math:`G : \mathcal D ‚Üí \mathcal C` together with natural isomorphisms :math:`Œµ : FG ‚âÖ \mathrm{id}_{\mathcal D}`, and :math:`Œ∑ : \mathrm{id}_{\mathcal C} ‚âÖ GF`. We say that :math:`F` is an equivalence with an inverse equivalence :math:`G` and denote the equivalence by :math:`F : \mathcal C ‚âÉ \mathcal D : G`.

    essentially surjective on objects
      A functor :math:`F : C ‚Üí D` is called **essentially surjective on objects** if for every object :math:`D ‚àà \mathcal D`, there is some :math:`A ‚àà \mathcal C` such that :math:`F A` is isomorphic to :math:`D`.

    existential image functor

    evaluation functor

    full functor
      A functor :math:`F : \mathcal C ‚Üí \mathcal D` is called **full** if for all objects :math:`A`, :math:`B` in :math:`\mathcal C`, the map :math:`\mathcal C(A, B) ‚Üí \mathcal D(F A, F B)` is surjective.

    functor
      A **functor** :math:`F : \mathcal C ‚Üí \mathcal D` consists of a function :math:`F_0` that maps objects of :math:`\mathcal C` to objects of :math:`\mathcal D` and a function :math:`F_1` that maps morphisms of :math:`\mathcal C` to morphisms of :math:`\mathcal D` such that :math:`F` preserves (co)domains of morphisms, identities, and compositions.

      Let :math:`ùêÜ_1 =(V_1, E_1)` and :math:`ùêÜ_2 = (V_2, E_2)` be graphs. We say that a pair of functions :math:`f=(f_v,f_e)` is a **graph morphism** from :math:`ùêÜ_1` to :math:`ùêÜ_2` provided :math:`f_v : V_1 ‚Üí V_2`, :math:`f_e : E_1 ‚Üí E_2`, and for any edge :math:`e = (v_1,v_2) ‚àà E_1` we have that we have :math:`f_e(e) = (f_v(v_1), f_v(v_2))`.

    group
      A **group** :math:`ùêÜ = (G, e, \ ^{-1}, ‚ãÜ)` consists of a set :math:`G` together with a nullary (constant) operation :math:`e`, a unary (inverse) operation :math:`\ ^{-1}: G ‚Üí G`, and a binary operation :math:`‚ãÜ : G^2 ‚Üí G`, such that :math:`(G, e, ‚ãÜ)` is a monoid and :math:`x ‚ãÜ x^{-1} = e` for all :math:`x ‚àà G`.

    Heyting algebra
      A **Heyting algebra** :math:`‚ü®L, ‚àß, ‚à®, ‚ä•, ‚ä§, ‚Üí‚ü©` is a bounded :term:`lattice` with least and greatest elements ‚ä• and ‚ä§, and a binary "implication" ‚Üí that satisfies :math:`‚àÄ a, b, c ‚àà L, \ (c ‚àß a ‚â§ b \ ‚ü∫ \ c ‚â§ a ‚Üí b)`.  Logically, this says a ‚Üí b is the weakest proposition for which the modus ponens rule, :math:`\{a ‚Üí b, a\} ‚ä¢ b`, is sound. The class of Heyting algebras forms a variety that is finitely axiomatizable.

    monoid
      A **monoid** :math:`ùêå = (M, e, ‚ãÜ)` consists of a set :math:`M` with a a **unit** element :math:`e ‚àà M` and a binary operation :math:`‚ãÜ : M^2 ‚Üí M` such that for all :math:`x,y,z ‚àà M`, :math:`x ‚ãÜ e = x = e ‚ãÜ x` and :math:`(x ‚ãÜ y) ‚ãÜ z = x ‚ãÜ (y ‚ãÜ z)`.

    monoid homomorphism
      Given monoids :math:`ùêå_1 = (M_1, e_1, ‚ãÜ)` and :math:`ùêå_2 = (M_2, e_2, ‚àó)` we say that a function :math:`f : M_1 ‚Üí M_2` is a **monoid homomorphism** from :math:`ùêå_1` to :math:`ùêå_2` provided :math:`f` preserves the nullary (identity) and binary operations; that is, :math:`f(e_1) = e_2` and :math:`f (x ‚ãÜ y) = f(x) ‚àó f(y)` for all :math:`x, y ‚àà M_1`.

    monomorphism
      A morphism :math:`f: A ‚Üí B` is called a **monomorphism** if for every object :math:`X` and every pair :math:`h, h' : X ‚Üí A` of morphisms, :math:`f ‚àò h = f ‚àò h'` implies :math:`h = h'`. When :math:`f` is a monomorphism we often say :math:`f` is "mono" and write :math:`f: A ‚Ü£ B`.

    monotone function
      Given posets :math:`‚ü®A, ‚â§·¥¨‚ü©` and :math:`(B, ‚â§·¥Æ)` we say that a function :math:`f: A ‚Üí B` is **monotone** from :math:`‚ü®A, ‚â§·¥¨‚ü©` to :math:`‚ü®B, ‚â§·¥Æ ‚ü©` when for any :math:`x, y ‚àà A` we have that :math:`x ‚â§·¥¨ y` implies that :math:`f(x) ‚â§·¥Æ f(y)`.

    natural isomorphism

    natural transformation

We shall write :math:`Œ± : F ‚áí G : \mathcal C ‚Üí \mathcal D` to indicate that Œ± is a natural transformation from :math:`F` to :math:`G`, where :math:`F, G : \mathcal C ‚Üí \mathcal D`.

    point

    polymorphic function

    poset

    predicative

    preorder

Categories

.. glossary::

    1
      The only object is :math:`0`; the only morphism is the identity :math:`\operatorname{id}_0: 0 ‚Ü¶ 0`.

    2
      There are two objects, :math:`0` and :math:`1`; there is one nonidentity morphism :math:`f: 0 ‚Ü¶ 1`.

    3
      There are three objects, :math:`0`, :math:`1`, and :math:`2`; there are three nonidentity morphisms: :math:`f: 0 ‚Ü¶ 1`, :math:`g: 1 ‚Ü¶ 2`, and :math:`h: 0 ‚Ü¶ 2`.

    Cat 
      the (large) category of small categories; it has small categories as objects and functors :math:`F : \mathcal C ‚Üí \mathcal D` as morphisms.

    Set
      the category whose objects are the sets and whose morphisms are the functions on sets.

    Grph
      the category whose objects are the (directed) graphs; the morphisms are the :math:`graph morphisms <graph morphism>`.

    Mon
      the category whose objects are the :term:`monoids <monoid>` and whose morphisms are the :term:`monoid homomorphisms <monoid homomorphism>`.

    Par
      the category whose objects are sets and whose morphisms are the :term:`partial functions <partial function>`.

    Rel
      the category whose objects are sets and whose morphisms are the :term:`relations <relation>` on sets.

    Fin
      a category whose objects are the finite sets; the morphisms are the functions on finite sets.

    Pos
      a category whose objects are the :term:`posets <poset>`; the morphisms are the :term:`monotone functions <monotone function>`.

    Lat
      a category whose objects are the :term:`lattices <lattice>`; the morphisms are the :term:`lattice homomorphisms <lattice homomorphism>`.

    CLat
      a category whose objects are the :term:`complete lattices <complete lattice>`; the morphisms are the :term:`complete lattice homomorphisms <complete lattice homomorphism>`.

    Arrow
      Given a category :math:`\mathcal C`, the **arrow category** :math:`\mathcal C^‚Üí` has as objects the triples :math:`(A, B, f)` satisfying :math:`A, B ‚àà  \mathcal C_{\mathrm{obj}}` and :math:`f ‚àà \mathcal C(A,B)`, and as morphisms the pairs :math:`(h_1, h_2) : (A, B, f) ‚Üí (C, D, g)` such that :math:`h_1 ‚àà \mathcal C(A,C)`, :math:`h_2 ‚àà \mathcal C(B, D)` and :math:`g \circ h_1 = h_2 \circ f`.

    Slice
      Given a category :math:`\mathcal C` and an object :math:`C ‚àà \mathcal C_{\mathrm{obj}} `, the **slice category** :math:`\mathcal C/C` has objects the pairs :math:`(A, f)` such that :math:`f ‚àà \mathcal C(A, C)`, and morphisms :math:`h : (A, f) ‚Üí (B, g)` such that :math:`h ‚àà \mathcal C(A, B)` and :math:`g ‚àò h = f`.

    Comma
      Given categories :math:`\mathcal C` and :math:`\mathcal D` and functors :math:`F : \mathcal C ‚Üí \mathcal D` and :math:`G : \mathcal C' ‚Üí \mathcal D` (with a common :term:`codomain`), the **comma category** is denoted by :math:`(F ‚Üì G)` and has objects the triples :math:`(A, f, A')`, where :math:`A ‚àà \mathcal C_{\mathrm{obj}}`, :math:`A' ‚àà \mathcal C'_{\mathrm{obj}}`, and :math:`f ‚àà \mathcal D(FA, GA')`, and morphisms the pairs :math:`(œÜ, œà) : (A, f, A') ‚Üí (B, g, B')`, where :math:`œÜ ‚àà \mathcal C(A, B)`, :math:`œà ‚àà \mathcal C'(A',B')` and :math:`G œà ‚àò f = g ‚àò F œÜ`.
